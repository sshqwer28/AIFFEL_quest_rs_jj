# PRT(Peer Review Template)
- [ ]  **1. 주어진 문제를 해결하는 완성된 코드가 제출되었나요?**
    - 1번, 2번 과제에 대한 코드는 잘 작성됨
    - 여러 케이스(회전, 옆모습, 축소 등)에 대해 테스트 진행함
    
- [ ]  **2. 전체 코드에서 가장 핵심적이거나 가장 복잡하고 이해하기 어려운 부분에 작성된 
주석 또는 doc string을 보고 해당 코드가 잘 이해되었나요?**
    - 설명이 필요한 부분에 주석으로 표시를 해둬서 이해 할 수 있었음
    - 각 과제에 해당되는 큰 제목과 주석으로 이해하기 쉽게했음
        
- [ ]  **3. 에러가 난 부분을 디버깅하여 문제를 해결한 기록을 남겼거나
새로운 시도 또는 추가 실험을 수행해봤나요?**
    - 얼굴 회전(yaw, roll)에 따른 스티커 회전을 적용시켰고, 해당 부분에 대한 주석 및 설명이 있어서 이해하기 쉬웠음
    - 위 기능으로 얼굴 각도가 회전해 있더라도 자연스럽게 스티커를 적용시킬 수 있음을 확인

- [ ]  **4. 회고를 잘 작성했나요?**
    - 코드를 알아보기 쉽게 잘 정리한 점이 좋았음
    - 기본적인 과제 외에 추가적인 기능을 넣은 것이 인상깊었음
    - 문제점 및 궁금증에 대한 내용이 정리돼 있음

        
- [ ]  **5. 코드가 간결하고 효율적인가요?**
    - 코드의 내용을 잘 알 수 있도록 주석, 변수명 등이 잘 작성돼 있음
    - 필요한 부분으로 잘 작성돼 있음


# 회고(참고 링크 및 코드 개선)
추가적인 과제를 수행한 점이 인상 깊었고, 좀 더 다듬으면 정말 좋은 결과물이 나올 수 있을 것 같음
